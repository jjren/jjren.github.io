我的Linux系统使用体验

发表于 2016-11-17

简单交代一下背景，我并不是学计算机出身的，只因为科研内容与计算科学相关，而计算科
学所使用的服务器绝大多数都是Linux系统，平时用的各种软件，也都是安装运行在Linux下
面。因此，平时少不了要和Linux打交道。

我的Linux使用经历大概分为下面三个阶段：
1.在windows下，使用Xmanager等软件，ssh远程登陆到计算服务器上使用，仅限于提交一下
任务，简单的数据处理，经常碰到网络延时等造成的烦恼；
2.在windows下，用vmware安装Linux虚拟机（我记得当时安装的是Redhat6.4）。解决了1中
网络延时的问题，可以自己安装一部分生产力工具。很多时候需要源码安装，也麻烦。
3.安装了ubuntu-14.04系统（Linux一个发行版），绝大多数工作都在ubuntu中进行，包括
一小部分的业余娱乐。当然还是保留了之前的window系统，主要是处理一些其他人发给我的
Office文档。

我从今年1月份的时候，才决心将日常的工作都放在Linux系统下进行。也算是用了将近一年
了，工作的幸福感有了很大的提升，但是刚开始的时候也确实遇到了一些不习惯，这里与大
家分享一些我的体会，以及一些我搜集到的Linux使用技巧。首先说明，我其实是一个技术
小白，不喜欢折腾，所以很多下面写到的攻略，技巧，都是拿来直接用，可以说是只知其然
，不知其所以然。我相信，使用Linux系统的人中，一部分也是和我一样情况的人。另外很
多下面分享的东西并不一定是最优解，也有可能有错误，也请各位大神不吝赐教。

ubuntu作为一个小白最好的入门Linux系统，它的包管理软件是其一大杀手锏。
1.安装软件:
一般的软件sudo apt-get install <package> 就可以搞定。其他特殊的软件，直接google
，记住你不是一个人。
2.删除软件:
sudo apt-get remove <package> && sudo apt-get autoremove
or
sudo aptitude remove <package>
http://askubuntu.com/questions/1143/how-can-i-uninstall-software
3.升级软件
sudo apt-get update
sudo apt-get upgrade

日常必备软件安装篇：（直接可以apt安装的我就不说明了，不能直接安装的我会给出方法
，或者给出方法的链接）

1.编辑器：vim
默认只提供了vim.tiny
(让我转到Linux的原因1，最主要原因)

2.显卡驱动：
https://help.ubuntu.com/community/BinaryDriverHowto/AMD

3.浏览器：chrome
http://askubuntu.com/questions/510056/how-to-install-google-chrome

8.Latex软件：Texlive2016
图形界面 apt-get install perl-tk
安装路径 默认的为/usr/local/texlive 改为 /opt/texlive
sudo chown yourname:yourname /opt/texlive
./install-tl -gui -repository \
http://mirrors.tuna.tsinghua.edu.cn/CTAN/systems/texlive/tlnet/
进去之后将文件夹改成/opt/texlive即可
http://zhuanlan.zhihu.com/LaTeX/20069414
http://tex.stackexchange.com/questions/287419/error-25-during-equivs-build-fake-packages-for-texlive2015
最后一步做虚拟包不用这么复杂，可以直接运行做好的包
texlive-local_2015-1_all.deb

16.中文输入法：搜狗输入法
先按照搜狗官网安装fcixt和sogou
http://pinyin.sogou.com/linux/help.php
然后配置的时候找到sogou-chinese加入

要在global config 的advance 里面将scroll between inactive选上
sogou输入法linux的缺点：没有同步词库的功能
(搜狗输入法解决了中文用户在Linux下面生存的一大难题)

21.ssh服务，让远程的计算机可以登陆

一、采用简单的密码登陆

  1. 设置openssh-server
    https://help.ubuntu.com/community/SSH/OpenSSH/Configuring
    命令
    sudo apt-get install openssh-server
    sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.factory-defaults
    sudo chmod a-w /etc/ssh/sshd_config.factory-defaults
    sudo restart ssh
    最好设置禁止root用户登陆PermitRootLogin no
    另外的参考
    http://songtl.com/linux-ssh-setting.html
    http://ihacklog.com/post/how-to-enable-secure-shell-in-ubuntu.html
  2. 设置防火墙，开启22端口
    sudo ufw allow 22
    https://help.ubuntu.com/lts/serverguide/firewall.html

二、采用更加安全的SSH-Key登陆
直接将public key放到远程的.ssh/authorized_keys文件夹中
这样可以实现不用密码就可以登陆
https://help.ubuntu.com/community/SSH/OpenSSH/Keys
命令
ssh-copy-id <username>@<host>

最后说说用了这么长时间，不顺手的地方:
1.其他人发过来的各种Microsoft 的word，ppt文件兼容性还不好，需要回到window下面修
改。自带的libreoffice有时候会卡死，需要时常记得保存。这一点MSOffice还是做的相当
不错的。MSOffice都出了Mac版了，不知道什么时候会出一个Linux版。
2.没有好用的阿里巴巴Linux版，所以一直用网页版。

我当初下定决心转到Linux，是因为vim，git，intel parallel studio等软件。一开始确实
会遇到各种各样的用不惯的问题，但是天天用，熟悉了以后，你会发现有很多的优势。相信
我，假如你是做计算科学的，Linux桌面版是你最好的选择！还能在业余时间，写写自己的
博客，何乐而不为呢！
